# Contributing

1. Fork the repo, create a branch: `git checkout -b feature/your-feature`
2. Install deps: `npm install`
3. Copy `.env.example` to `.env`
4. Run dev server: `npm run dev`
5. Make your change, commit, push, open a PR
6. Link the issue your PR resolves

Look for issues labeled `good first issue` if you're new here.

## Code style

- ES modules (`import`/`export`)
- Routes in `src/routes`, logic in `src/controllers`, data in `src/models`
- Open a discussion in the issue before large structural changes (e.g. choosing a database)
